# Data files

## What's what

| File | Contains | Committed? | Notes |
|------|----------|------------|-------|
| `public/data.json` | Real sales data (frontend) | **NO** (`.gitignore`) | Generated by `tools/csv-to-json.js` |
| `public/data.sample.json` | Fake demo data | Yes | App falls back to this automatically |
| `tools/data.json` | Real sales data (intermediate) | **NO** (`.gitignore`) | Generated by `tools/csv-to-json.js` |
| `tools/data.sample.json` | Fake demo data | Yes | Reference for the tools pipeline |
| `tools/*.csv` | Real source CSVs | **NO** (`.gitignore`) | Your raw sales exports |
| `tools/csv-to-json.js` | Conversion script | Yes | Code, not data |

## How the app loads data

1. Tries to fetch `/data.json` (your real data).
2. If missing or empty, falls back to `/data.sample.json` (demo data).
3. Header shows "Sample data" when using the fallback.

## Generating real data

```bash
# Place your CSV in tools/, then:
node tools/csv-to-json.js
# Creates tools/data.json AND public/data.json
```

## First-time setup after cloning

The repo has no real data. The app will load `data.sample.json` automatically.
To use your own data, place your CSV in `tools/` and run the command above.
